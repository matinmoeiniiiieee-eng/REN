# 🚀 REN Gateway

تونل VLESS over WebSocket با پنل مدیریت حرفه‌ای

---

## 📌 نحوه کارکرد

REN Gateway یک تونل VLESS روی WebSocket است که روی Render یا Railway (هاست رایگان) اجرا می‌شود.

نیازی به سرور VPS ندارید. فقط کافیه پروژه را فورک کنید و روی Render یا Railway دیپلوی کنید.

بعد از دیپلوی، یک آدرس بهتون داده می‌شود که پنل مدیریت شماست.

### خلاصه نحوه استفاده:

- 📱 اپلیکیشن v2rayNG یا NekoBox را نصب کنید
- 🔗 لینک VLESS را از پنل کپی کنید
- 📲 لینک را در اپلیکیشن وارد کنید
- ✅ اتصال برقرار شد!

---

## ⚡ شروع سریع (Render)

### ۱. فورک کنید
- روی دکمه **Fork** بالای این صفحه کلیک کنید

### ۲. به Render بروید
- به [render.com](https://render.com) بروید
- با اکانت GitHub وارد شوید
- روی **New** → **Web Service** کلیک کنید
- مخزن فورک شده خود را انتخاب کنید

### ۳. تنظیمات دیپلوی
- **Build Command:** `pip install -r requirements.txt`
- **Start Command:** `python main.py`
- **Name:** هر اسمی که دوست دارید
- **Region:** Frankfurt (Germany) برای پینگ بهتر
- ⚠️ **Plan:** حتماً **Free** را انتخاب کنید
- 🔑 **رمز عبور:** در بخش **Environment** یک متغیر به نام `ADMIN_PASSWORD` با یک رمز **قوی (حداقل ۸ کاراکتر)** بسازید. دیگر رمز پیش‌فرضی داخل کد وجود ندارد.

### ۴. دیپلوی کنید
- روی **Deploy** کلیک کنید
- ⏳ حدود ۲-۳ دقیقه صبر کنید

### ۵. وارد پنل شوید
- بعد از دیپلوی، Render یک آدرس مثل این بهتون می‌ده:
  ```
  https://your-app-name.onrender.com
  ```
- آدرس پنل شما:
  ```
  https://your-app-name.onrender.com/login
  ```
- 🔑 با رمزی که در `ADMIN_PASSWORD` تنظیم کردید وارد شوید.
- ⚠️ اگر `ADMIN_PASSWORD` را تنظیم نکرده باشید، پنل با رمز موقت `admin` بالا می‌آید و در اولین ورود، یک هشدار قرمز نمایش می‌دهد و شما را مجبور به تغییر رمز می‌کند.
- 🛡️ پس از ۵ تلاش ناموفق برای ورود، آی‌پی شما به‌طور موقت قفل می‌شود (محافظت در برابر brute-force).

⚠️ **نکته مهم:** فقط Render از آی‌پی‌های Cloudflare پشتیبانی می‌کند. اگر می‌خواهید از آی‌پی تمیز Cloudflare استفاده کنید، حتماً Render را انتخاب کنید.

## ⚡ شروع سریع (Railway)

### ۱. فورک کنید
- روی دکمه **Fork** بالای این صفحه کلیک کنید

### ۲. به Railway بروید
- به [railway.com](https://railway.com) بروید
- با اکانت GitHub وارد شوید (دقیقاً مثل Render)
- روی **New Project** → **Deploy from GitHub Repo** کلیک کنید
- مخزن فورک شده خود را انتخاب کنید
- روی **Deploy** بزنید و منتظر بمانید تا تمام شود

### ۳. تنظیمات ریجن
- بعد از دیپلوی، روی پروژه کلیک کنید
- به بخش **Settings** بروید
- قسمت **Scale** را پیدا کنید
- ریجن را روی **Amsterdam** تنظیم کنید تا پینگ بهتری داشته باشید

### ۴. وارد پنل شوید
- بعد از دیپلوی، Railway یک آدرس مثل این بهتون می‌ده:
  ```
  https://your-app-name.up.railway.app
  ```
- آدرس پنل شما:
  ```
  https://your-app-name.up.railway.app/login
  ```
- 🔑 در Railway هم یک متغیر `ADMIN_PASSWORD` با رمز قوی بسازید (Variables). اگر تنظیم نشود، رمز موقت `admin` است و باید در اولین ورود تغییرش دهید.

## 🎯 آی‌پی تمیز (Clean IP)

### آی‌پی تمیز چیست؟
آی‌پی تمیز یک IP یا دامنه است که می‌تواند پینگ بهتری برای شما فراهم کند. استفاده از آی‌پی تمیز اختیاری است ولی توصیه می‌شود.

### چطور استفاده کنیم؟
1. وارد پنل شوید
2. بخش **Clean IP** را باز کنید
3. آدرس‌های دلخواه خود را اضافه کنید (هر تعداد که بخواهید)
4. کانفیگ‌ها خودکار با آی‌پی‌های تمیز ساخته می‌شوند

⚠️ **نکته مهم:** فقط Render از آی‌پی‌های Cloudflare پشتیبانی می‌کند. Railway از Cloudflare IPs پشتیبانی نمی‌کند.

## 🔄 تفاوت کپی و سابسکریپشن

### دکمه Copy (کپی)
- فقط کانفیگ با **دامنه خود سرور** را کپی می‌کند
- برای استفاده سریع و تست اتصال مناسب است

### لینک سابسکریپشن (Subscription)
- **تمام کانفیگ‌ها** شامل دامنه سرور + تمام آی‌پی‌های تمیز را کپی می‌کند
- ✅ نمایش حجم مصرفی و زمان باقی‌مانده در اپلیکیشن
- ✅ آپدیت خودکار کانفیگ‌ها بدون نیاز به کپی مجدد
- ✅ مناسب برای استفاده طولانی‌مدت
- ✅ مدیریت آسان‌تر چندین اتصال

## 🎯 استفاده از پنل

### ساخت لینک VLESS
1. وارد پنل شوید
2. روی **+ Add** کلیک کنید
3. یک نام انگلیسی وارد کنید (مثلاً: `Ali`)
4. حجم مصرفی را مشخص کنید (مثلاً: `1` GB)
5. روی **Create** کلیک کنید
6. لینک VLESS را کپی کنید یا QR Code را اسکن کنید

### اتصال با اپلیکیشن
1. اپلیکیشن **v2rayNG** یا **NekoBox** را نصب کنید
2. لینک VLESS را وارد کنید
3. اتصال را فعال کنید
4. 🎉 تمام! حالا می‌توانید از اینترنت آزاد استفاده کنید

### مدیریت لینک‌ها
- 🔍 **جستجو:** در بخش Inbounds می‌توانید لینک‌ها را جستجو کنید
- 🏷️ **فیلتر:** لینک‌ها را بر اساس وضعیت (فعال/غیرفعال) فیلتر کنید
- ⚡ **فعال/غیرفعال:** با کلیک روی دکمه Toggle لینک را فعال یا غیرفعال کنید
- 📊 **مصرف:** مصرف حجم هر لینک را ببینید
- ✏️ **ویرایش:** با کلیک روی دکمه Edit حجم و IP لیمیت را تغییر دهید
- 🗑️ **حذف:** لینک‌های بدون استفاده را حذف کنید

## ⚙️ متغیرهای محیطی (Environment Variables)

| متغیر | پیش‌فرض | توضیح |
|-------|---------|-------|
| `ADMIN_PASSWORD` | `admin` (موقت) | رمز ورود به پنل. حتماً یک رمز قوی تنظیم کنید |
| `SECRET_KEY` | خودکار روی Render | کلید داخلی؛ روی Render خودکار ساخته می‌شود |
| `ALLOW_PRIVATE_RANGES` | `false` | اگر `true` شود، تونل اجازه اتصال به IPهای داخلی/loopback را می‌دهد (توصیه نمی‌شود) |
| `DATA_FILE` | `data/ren_data.json` | مسیر فایل ذخیره‌سازی داده‌ها |
| `PORT` | `8000` | پورت؛ روی Render/Railway خودکار تنظیم می‌شود |

## 🔒 امنیت

نسخهٔ فعلی چند بهبود امنیتی مهم دارد:

- 🔑 **هیچ رمز پیش‌فرضی داخل کد نیست.** رمز از `ADMIN_PASSWORD` خوانده می‌شود و اگر تنظیم نشود، پنل شما را مجبور به تغییر آن می‌کند.
- 🧂 رمز با **PBKDF2-HMAC-SHA256** (۲۰۰٬۰۰۰ تکرار) و salt تصادفی هش می‌شود و مقایسه به‌صورت constant-time انجام می‌گیرد.
- 🛡️ **محافظت brute-force:** پس از ۵ تلاش ناموفق، ورود از آن آی‌پی به‌مدت ۱۵ دقیقه قفل می‌شود.
- 🚫 **محافظت SSRF:** تونل به‌طور پیش‌فرض اجازهٔ اتصال به رنج‌های داخلی و متادیتای ابری (مثل `169.254.169.254`) را نمی‌دهد.
- 🍪 کوکی سشن با `HttpOnly` و روی HTTPS با `Secure` تنظیم می‌شود.

## 💾 پایداری داده‌ها (Persistence)

داده‌ها (لینک‌ها، آی‌پی‌های تمیز، دامنه، رمز) در یک فایل JSON (`data/ren_data.json`) ذخیره و هنگام اجرا بارگذاری می‌شوند.

⚠️ **توجه مهم دربارهٔ هاست رایگان:** فایل‌سیستم پلن رایگان Render و Railway **موقت (ephemeral)** است. یعنی داده‌ها بین ری‌استارت‌های درون‌سرویسی می‌مانند اما با **ری‌دیپلوی یا cold start پاک می‌شوند**. برای پایداری کامل:

- از دکمهٔ **Export/Import** در بخش System برای بکاپ دستی استفاده کنید، یا
- یک **دیسک پایدار** (پلن پولی Render) روی مسیر `./data` وصل کنید، یا
- یک دیتابیس خارجی (مثل Redis/Postgres رایگان) وصل کنید — این مورد در فاز بعدی قابل افزودن است.

## 🧪 توسعه و تست

اجرای محلی:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements-dev.txt
pytest -q          # اجرای تست‌ها
python main.py     # اجرای سرور
```

پروژه شامل مجموعه‌ای از تست‌ها (pytest) برای پارس پروتکل VLESS، منطق سهمیه، هش رمز، محدودیت نرخ ورود و APIهای اصلی است.
